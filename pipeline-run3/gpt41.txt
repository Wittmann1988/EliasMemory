## SECTION 3: BUILD STEPS

1. **Clone repository:**
   ```
   git clone https://github.com/yourorg/ai-pipeline.git
   cd ai-pipeline
   ```

2. **Set up Python environment (recommended: `venv`):**
   ```
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   ```

3. **Set up Node.js (for Android/Termux client):**
   ```
   cd android-client
   npm install
   ```

4. **Set up SQLite-vec and DB:**
   ```
   python -m sqlite_vec.cli init --db ./data/vector_store.db --dim 2048
   ```

5. **Build and start API server (for pipelines):**
   ```
   uvicorn src.api.fastapi_app:app --host 0.0.0.0 --port 8000
   ```

6. **(Optional) Use Docker Compose for prod:**
   ```
   docker-compose up --build
   ```

## SECTION 4: EVENT BUS PROTOCOL

### Goal

- Decoupled, language-agnostic communication between components, locally and over network.

### Transport

- *Preferred:* Intra-process: Python `asyncio` + in-memory events.
- *Inter-device:* HTTP (REST & WebSocket). Use FastAPI for API; sockets for streaming events.
- *Message serialization:* JSON for basic, Protobuf or MsgPack for high-volume.

### Message Schema (YAML/JSON/Protobuf):

```yaml
# event_schema.yaml
type: string                   # e.g., "MEMORY_UPDATE", "TASK_REQUEST"
event_id: uuid
timestamp: iso8601
origin: string                 # "android-tablet", "jetson", "desktop", etc.
target: string                 # destination
payload: object                # body (see below)
meta:                          # optional
  priority: int
  trace_id: uuid
  tags: [string]
```

#### Example (JSON):

```json
{
  "type": "TASK_REQUEST",
  "event_id": "f3d2b95c-9f8b-4efd-a626-20240421T1530Z",
  "timestamp": "2024-06-22T14:00:00Z",
  "origin": "android-tablet",
  "target": "jetson",
  "payload": {
    "task": "summarize_document",
    "doc_id": "c5c3f3d",
    "parameters": {
      "max_tokens": 500,
      "language": "en"
    }
  },
  "meta": {
    "priority": 3,
    "trace_id": "b9bb-f3e1-234b",
    "tags": ["urgent"]
  }
}
```

#### Standard Event Types

- `TASK_REQUEST` (assign work)
- `TASK_RESULT` (send result)
- `MEMORY_UPDATE` (graph/memory action: add, modify, delete)
- `EMBEDDING_REQUEST` / `EMBEDDING_RESULT`
- `AGENT_STATE_UPDATE`
- `PIPELINE_STEP_START`, `PIPELINE_STEP_END`
- `SELF_IMPROVEMENT_EVENT`
- `ERROR`

See `src/event_bus/protocol.py` for full schema classes.

## SECTION 5: SELF-IMPROVEMENT LOOP

### Mechanisms

Improvements via DSPy + Mem0/A-MEM, with RL where appropriate.

**Algorithm:**
1. **After each pipeline execution:**
    - Capture full context: inputs, outputs, intermediate states, evaluation metrics (ground-truth if available, else heuristic, else ranking).
2. **Store run metadata:**
    - Commit fields to Mem0/A-MEM as 'run nodes,' linked to the graphs.
3. **Trigger DSPy self-improvement:**
    - Update `dspy_interface.py`:
        - Call `dspy.optimize()` with run record, feedback, metric.
        - Reinforce or adjust agent/pipeline config/primitives.
4. **Memory Agents (A-MEM, AgentFly):**
    - RL module assigns credit/blame to memory/graph elements that contributed to outcome.
    - Letta/MemGPT dual-paging: monitor what gets paged in/out during runs, bias future access.
5. **Consensus/Meta-agent:**
    - Write improvement event to event bus:
        ```
        type: SELF_IMPROVEMENT_EVENT
        payload: {
            "component": "orchestrator/agent/memory",
            "improvement": "hyperparameter update",
            "details": {...}
        }
        ```
6. **[Optional, nightly] Replay:** 
    - Re-run recent samples with updated pipeline; log regression/progress; rollback if metrics degrade.

### Self-Improvement Loop Skeleton
(Python, in `pipeline/improvement.py`)
```python
def self_improve(run_context: dict, metrics: dict) -> dict:
    """
    After a completed pipeline run, analyze and propose improvements.
    run_context: inputs/outputs/intermediates
    metrics: evaluation (score, latency, etc)
    Returns dict of proposed modifications.
    """
    # Example: Use DSPy
    improvement = dspy.optimize(
        last_run=run_context,
        feedback=metrics,
        memory_graph=mem0.current_state()
    )
    # Post improvement event to event bus
    event_bus.post({
        "type": "SELF_IMPROVEMENT_EVENT",
        "payload": improvement
    })
    # Update pipeline if accepted
    apply_improvement(improvement)
    return improvement
```

## SECTION 7: PHASED ROLLOUT PLAN

### PHASE 1: **MVP CORE PIPELINE**
- Goal: End-to-end functional pipeline; simple memory; basic agent; embedding.
- **Build:**
    - Event bus (core, protocol)
    - FastAPI API server (`api/`)
    - LangGraph pipeline orchestrator (`orchestrator.py`)
    - sqlite-vec interface + Llama embeddings
    - Android/Node.js thin client (can submit requests, retrieve results)
- **Test:** Basic end-to-end from Android to Jetson to Desktop and back.

### PHASE 2: **ADVANCED MEMORY & AGENTS**
- Goal: Add graph memories (Mem0, A-MEM), basic CrewAI agent teams.
- **Build:**
    - Integrate Mem0 in-memory graph
    - Integrate A-MEM (Zettelkasten)
    - Build CrewAI agents and task delegation
    - Enable memory paging with Letta/MemGPT on Jetson/GPU node
    - Start self-improvement loop via DSPy; basic metrics
- **Test:** Contextual tasks; memory retrieval/improvement.

### PHASE 3: **FULL SELF-IMPROVEMENT & RL**
- Goal: Dynamic, adaptive pipeline with RL-based memory construction and pipeline optimization.
- **Build:**
    - AgentFly/Mem-alpha RL module integration
    - Full run-logging, feedback, and replay system
    - Consensus/Meta-agent for architecture improvements
    - Allow rollback/upgrades against history
- **Test:** Run hundreds of iterations; demonstrate learning/retuning; regression test.
